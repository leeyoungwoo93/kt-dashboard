# v5-seeded-retail-monthly

## 수정 핵심
- v4에서 판매 데이터가 안 뜬 직접 원인: 배포 안정화 명목으로 엑셀 자동 적재를 끄고 데이터 파일도 제외해서 운영 DB가 빈 상태였다.
- v5는 `app/seed/kt_dashboard_seed.db`를 포함한다.
- 기본 SQLite DB(`kt_dashboard.db`)가 없거나 1MB 미만의 빈 DB이면 앱 시작 전에 seed DB를 자동 복사한다. 필요 시 `FORCE_SEED_DB=1`로 강제 재복사할 수 있다.
- 판매 원천의 `본부 코드 + 본부명`, `담당 코드 + 담당명`, `무선유통조직 코드 + 명칭` 구조를 코드가 아니라 명칭 기준으로 읽도록 보정했다.
- `/api/health`에 판매/소매 행수, 판매 합계, 기준월 목록을 반환하도록 확장했다.
- 화면 상단에 실제 로드된 판매 행수와 기준월을 표시한다.

## 포함 데이터
- 판매: 84,263행 / 판매량 1,491,685
- 소매 매장실적: 186,296행 / 판매량 555,495
- 기준월: 202504, 202601, 202602, 202603, 202604

## 배포 후 확인
1. Railway Deployment가 Success인지 확인
2. `/api/health` 접속
3. `sales_rows > 0`, `sales_total > 0`이면 데이터 로드 정상
4. 화면 상단 배지가 `v5-seeded-retail-monthly`로 보여야 함
